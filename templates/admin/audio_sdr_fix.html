Audio/SDR Configuration Fix

About This Tool

This page helps diagnose and fix three common audio/SDR issues:

  1. MP3 streams not appearing as Icecast mount points
  2. Waterfall display showing wrong frequency scale (kHz instead of MHz)
  3. High-pitched squeal/tone from SDR audio

Root Cause: Sample rates misconfigured (audio rate used for IQ samples)

Step 3: Restart Services

Configuration has been updated! Now restart the SDR service to apply the changes.

Run this command on your Raspberry Pi:

sudo docker compose -f docker-compose.yml -f docker-compose.pi.yml restart sdr-service

After restarting, check:

  • Icecast mount points: http://localhost:8001/
  • Waterfall display: /settings/radio
  • Listen to streams - should be clear audio, no squeal
