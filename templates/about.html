About the KR8MER CAP Emergency Alert System

This community-driven project distributes National Weather Service CAP alerts, SAME audio, and LED board messaging for regional emergency communications partners.

The platform couples a Flask web application with scheduled ingest services, Azure-powered voice synthesis, and hardware integrations so local operators can monitor, validate, and relay critical weather and civil alerts.

System Architecture Overview

  • Data Acquisition: NOAA CAP feeds are polled on a schedule, parsed, and persisted with full geometry metadata for downstream analysis.
  • Web Operations: Flask serves the operator dashboard, alert archives, statistics, LED control, and this public help content.
  • Broadcast Pipeline: The EAS encoder produces SAME headers, digital data bursts, and optional Azure AI Speech narration for broadcast-ready audio.
  • Hardware Integrations: Network LED signage and GPIO-triggered transmitters receive synchronized alert status updates.

Software Stack

The project combines open source geospatial tooling, broadcast utilities, and optional cloud services:

Core Platform

  • Python 3 with Flask & SQLAlchemy
  • GeoAlchemy2 & PostGIS spatial extensions
  • Celery-style pollers & scheduler helpers
  • Requests, Pydantic utilities, and PyTZ

Infrastructure

  • Docker & Docker Compose orchestration
  • PostgreSQL with PostGIS (dedicated container)
  • Redis-backed task queues (optional)
  • Raspberry Pi and networked LED controllers

Broadcast Chain

  • Custom SAME encoder & audio pipeline
  • SoX/FFmpeg utilities for waveform processing
  • Azure AI Speech synthesis (optional voiceover)
  • GPIO relay management scripts

Operator Tooling

  • Web dashboards & health monitors
  • CLI ingestion & manual activation scripts
  • Template repair utilities
  • Extensive documentation in ABOUT.md & HELP.md
